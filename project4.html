API's with Amazon
Web Services

Overview

This is an internship project I had where I needed to create an api that communicated with the front-end of one of our websites. For example, one of the website features was managing employees and their onboarding/offboarding tasks they needed to do. I worked on the design of the backend with my project manager, allocated all the resources, and programmed all the lambdas.

Issue At Hand

Our website had a front-end developed, but all our data we were using for our tables was hard-coded. We also needed the ability to add, remove, and update data in our tables.

Project Outcome

API methods were developed to perform full CRUD operations on our databases. No more need to hard-code data into our tables for testing.

Technologies used

  • Lambda Proxies
  • DynamoDB
  • API Gateway
  • Secrets Manager
  • Relational Database Services

what I Learned

  • How to build APIs using API Gateway and Lambda Proxy integration
  • How perform CRUD operations on relational databases
  • How communicate between the front-end and back-end of an application
