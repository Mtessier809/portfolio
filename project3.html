Process Daily
Checks

Overview

This is an internship project I had where I needed to retreive process data from client databases and send daily automated emails. Each client received their own email with a table that included all the process names and statuses on whether they were successful or not.

Current Issue

Our company had no way to track the statuses of processes for each client database. Because of this it was very difficult to pinpoint where something went wrong. The only way to do so was to look inside of the database and manually look for the problem.

Project Outcome

The daily checks queried all the new data entered into the databases each day and sent a nicely formatted table with all the process names and statuses. This helped pinpoint where issues were occurring and eliminated the need to manually check in databases.

  • DynamoDB
  • Simple Email Services
  • Secrets Manager
